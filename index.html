La inteligencia artificial

El campo de la IA

Machine Learning

Machine Learning es una rama de la inteligencia artificial que se ocupa de que las máquinas aprendan a partir de datos.

Una rama más avanzada es el deep learning, que utiliza redes neuronales con muchas capas, capaces de aprender patrones complejos de forma jerárquica: formas visuales, estilos o significados del lenguaje.

Muchas áreas usan técnicas de ML o DL para funcionar:

  • → Cuando una IA "ve": Computer Vision + Deep Learning
  • → Cuando una IA "traduce": NLP + Machine Learning
  • → Cuando genera imágenes desde texto: Visión + Lenguaje + DL

Redes Neuronales Artificiales
Redes Neuronales Artificiales

Una red neuronal artificial enseña a las computadoras a procesar datos inspirada en la forma en que lo hace el cerebro humano. Igual que las neuronas, son capaces de recibir y enviar señales.

Un grupo interconectado de nodos. Cada nodo representa una neurona y cada flecha una conexión. El comportamiento se controla a través de los parámetros.

3 partes:

  • → Entradas: números que recibe
  • → Pesos de cada entrada
  • → Salida: el número final obtenido

Realiza una serie de sumas y una función de activación.

LLM · Texto
En los LLMs el lenguaje natural es la interfaz principal: no hay parámetros para configurar.
  • Ser específico es mejor siempre que ser general.
  • Dar contexto mejora el resultado. El modelo no sabe quién sos ni para qué usás lo que genera. Cuanto más contexto relevante, mejor responde.
  • Definir el formato deseado.
  • Indicar el rol o perspectiva.
  • Iterar. El primer prompt rara vez es el mejor.
Un LLM no lee palabras: lee tokens.
Un token es un fragmento de texto que puede ser una palabra completa, parte de una palabra, un signo de puntuación o un espacio.
  • Los modelos tienen un límite de tokens por conversación (ventana de contexto). Cuando se supera ese límite, el modelo empieza a "olvidar" lo que se dijo al principio.
  • El costo de uso de APIs se mide en tokens: cuántos se reciben y cuántos se generan.
  • Los tokens se consumen tanto en el codificación del texto de entrada como en la decodificación al texto de salida.
Alucinación y límites
Verificar siempre los LLMs pueden alucinar con confianza. Fechas, citas, nombres, estadísticas, referencias bibliográficas: siempre verificar antes de usar.

Esto no es una falla ocasional sino una característica estructural de cómo funcionan estos modelos.

Estrategias de conversación
Roleplay y contexto
Definirle quién es, qué tiene que hacer, su tono. Indicar contexto, darle un objetivo, especificar un formato de respuesta. Decirle también lo que NO tiene que hacer.
Chain of thought (cadena de pensamiento)
Pedirle al modelo que razone paso a paso antes de dar una respuesta mejora significativamente la calidad en tareas complejas. Frases como "pensá paso a paso" o "antes de responder, identificá los supuestos implícitos" fuerzan cadenas de pensamiento.
Few-shot prompting
Dar ejemplos de lo que se espera antes de hacer el pedido. En lugar de describir el formato deseado, mostrárselo: "Acá hay dos ejemplos del estilo que busco... ahora generame uno similar sobre X".
Ventana de contexto
Cada LLM tiene un límite de tokens que puede procesar en una conversación, llamado ventana de contexto. Cuando la conversación es muy larga, el modelo puede "olvidar" lo que se dijo al principio. Para tareas largas conviene empezar una conversación nueva con el contexto relevante resumido.
Compartir archivos y links como fuentes
Incluir archivos de texto, imágenes y links permite contextualizar y focalizar en determinada información.
Deep research
Algunos LLMs ofrecen modos de investigación profunda, tomándose más tiempo para investigar antes de responder.
Nivel de pensamiento
Distintos planes de pago permiten acceder a modelos de niveles de "pensamiento" elevados capaces de resolver con precisión problemas matemáticos y físicos complejos.

Imagen · Difusión
¿Cómo nos comunicamos con un modelo generativo de imágenes?

Aquí el prompt es una instrucción estructurada de texto.

La clave para una comunicación exitosa con los sistemas de IA reside en la elaboración de un prompt bien estructurado.

En su construcción es clave el orden en los que se escriben los bloques: los primeros tendrán más peso y por ende son los que definirán nuestro pedido.

Cada palabra clave que escribamos tiene asociaciones no deseadas que tenemos que descubrir.

Debe ser detallado y específico.
Parámetros para esculpir la imagen
Seed [semilla]
Valor aleatorio a partir del que se genera el ruido inicial del espacio latente y, por lo tanto, la composición general de la imagen. Establecerlo en -1 lo hace aleatorio, para consistencia fijar en cualquier otro valor.
CFG [nivel de obediencia]
Classifier-Free Guidance es el parámetro más conversacional, ya que controla cuánto debe respetar el prompt.
muy bajo: ignora el prompt · bajo: muy creativo · medio: equilibrio entre creatividad y prompt · alto: adhiere más al prompt · muy alto: sigue estrictamente el prompt.
Evitar extremos: si es muy alto se sobresatura y quema la imagen por sobre-obedecer, mientras que si es muy bajo hace cualquier cosa.
Sampler
Cada sampler implementa un método distinto de eliminar el ruido.
· Euler A [rápido y más contrastado]
· DPM++ 2M Karras [más suave y detallado]
· DDIM [recomendado para img2img]
Steps [pasos de muestreo]
Cada paso de muestreo revisa y refina la imagen, con demasiados se sobreprocesa y la imagen ya no cambia. Recomendación general: 20-30 pasos. El número de pasos "adecuado" depende del Sampler elegido.
Denoising strength [reducción de ruido]
Cuando se trabaja a partir de otra imagen (img2img, inpaint, etc), es el nivel de ruido extra que le agrega a la imagen de entrada.
0.0 El modelo no añade ruido.
0.5 Modifica detalles medianos.
1.0 Borra por completo la imagen base.
tipo de contenido
qué tipo de output queremos, una foto, dibujo, renderizado 3D, etc.
+ descripción
sujeto, atributos y contexto. Cuanto más descriptivo, más preciso.
+ estilo
está conformado por luces, estilos o técnicas artísticas.
+ composición
la imagen en relación a la posición de la cámara y el tipo de lente.
+ prompt negativo
lo que no queremos ver.

LoRAs
A partir de un dataset de imágenes se pueden generar modificadores que se acoplan al modelo base para inyectarle un estilo, personaje o dirección de arte exacto sin alterar todo el modelo. Ideales para mantener una imagen consistente a lo largo de distintas iteraciones.

Permite guiar la composición de la imagen mediante mapas de profundidad, detección de bordes (Canny) o la pose exacta de un esqueleto humano (OpenPose).

Herramientas

Dónde encontramos los modelos

La elección entre plataforma privada y abierta implica decisiones sobre control, privacidad, costo y posibilidades técnicas. Cada modelo de IA tiene su propia moral y lo legal, y ambos responden a los intereses de la empresa que lo creó.

● Plataformas privadas online
Acceso por suscripción. Sin instalación. La empresa controla el modelo y los datos.

ChatGPT, Midjourney, Runway, ElevenLabs.
● Descarga y ejecución local
Modelos open source. Requiere GPU dedicada. Cada modelo pesa 2–20GB.

Flux, SD, Llama, Whisper vía ComfyUI u Ollama.
● Integrados en software
IA nativa en herramientas de diseño y productividad. Sin configuración técnica.

Adobe Firefly, Adobe Podcast, Descript.
